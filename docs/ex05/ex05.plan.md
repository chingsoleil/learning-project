# mockup畫面製作
* 整理資料結構
* 訂定頁面操作流程
* 訂定mockup頁面技術規格
* 依據user story 和資料結構建立範例資料
* 建立可操作的mockup頁面

## 頁面架構及操作流程
* mockup頁面須以手機操作為優先的RWD頁面
* 前端網頁框架：vue3、tailwind css
* 欄位名稱需參考資料字典，假資料以json格式儲存。

### 使用者受測頁面
#### 首頁
* 網站簡介：網站內容、各測驗選單及介紹

#### 單項測驗介紹 
* 測驗名稱、測驗簡介(文字及圖片)
* (以大五人格測試為例)
  * 簡化為50題，請出範例題目。
  * 存成json格式，依照資料欄位格式規劃。

#### 受測者輸入個人資訊
* 電話號碼
* e-mail
* 姓名
* 性別
* 年齡

#### 進行答題
* 題目列表
* 答案選擇
* 上頁/下頁/送出 按鈕

#### 完成答題/查看分數及初步分析
* 結果標題(同)
* 結果簡介(同)
* 個人資訊(異)
* 個人評分(異)
* 個人圖表(異)
  * 各種特質面向及分數
* 個人分析文字(異)

#### 查看詳細報告
* 受測者驗證：電話號碼/e-mail
* 選擇受測歷史資訊
* 查看詳細報告
* 諮詢表單：
  * 標題
  * 類別：(請補上3個常用諮詢類型)
  * 內容
  * 聯絡方式
  * 聯絡時間
